# 闲鱼商品监控器 APK

一个安卓应用，用于管理和轮询访问多个闲鱼商品页面，支持 **WebView 嵌入**、**URL 变化检测**、**自动化购买流程**、**购买限制自动重试**和**悬浮窗快捷控制**。

## 功能特点

- ✅ **商品 ID 管理** - 添加/删除/编辑多个商品 ID
- ✅ **批量访问** - 一键打开所有商品页面
- ✅ **自动轮询** - 定时循环访问商品（可配置间隔）
- ✅ **本地存储** - 商品列表自动保存，重启不丢失
- ✅ **批量输入** - 支持逗号/换行分隔批量添加
- ✅ **账号类型选择** - 个人号/企业号两种购买流程
- ✅ **WebView 嵌入** - 应用内直接加载页面，无需跳转浏览器
- ✅ **URL 变化检测** - 自动识别页面类型（商品页→订单页→付款页）
- ✅ **自动化购买** - 根据页面类型自动执行对应操作
- ✅ **购买限制检测** - 检测"无法购买"提示
- ✅ **自动重试** - 限制后等待 10 分钟自动重试（可配置）
- ✅ **循环次数设置** - 可配置轮询循环次数和间隔
- ✅ **悬浮窗控制** - 系统级悬浮窗，一键开关自动化

## 使用场景

- 监控多个商品的价格变化
- 定期检查商品是否下架
- 批量查看收藏夹商品
- 竞品监控
- **自动化抢购**（需谨慎使用）

## 核心功能说明

### 1. URL 变化检测

应用会自动识别当前页面类型：

| 页面类型 | URL 模式 | 图标 |
|----------|---------|------|
| 📦 商品页 | `2.taobao.com/item.htm?id=xxx` | 自动点击"立即购买" |
| 📋 订单页 | `buy.taobao.com/order` | 自动点击"确认购买" |
| 💰 付款页 | `cashier.alipay.com` | 自动点击"支付" |
| 🏠 首页 | `2.taobao.com` | 无操作 |

**优势：**
- 不依赖固定延迟，响应更快
- 页面跳转后立即触发操作
- 避免重复点击或错过步骤

### 2. 自动化购买流程

#### 个人号流程
```
📦 商品页 → 检测到 URL 变化 → 自动点击"立即购买"
         ↓
    检测购买限制
         ↓
📋 订单页 → 检测到 URL 变化 → 自动点击"确认购买"
         ↓
💰 付款页 → 检测到 URL 变化 → 自动点击"免密支付"
```

#### 企业号流程
```
📦 商品页 → 检测到 URL 变化 → 自动点击"立即购买"
         ↓
    检测购买限制
         ↓
💰 付款页 → 检测到 URL 变化 → 自动点击"免密支付"
```
（跳过订单确认页）

### 3. 购买限制检测与重试

**检测内容：**
- "亲，暂时无法购买"
- "暂时无法购买"
- "无法购买"
- "购买失败"
- "操作太频繁"
- "请稍后再试"
- "系统繁忙"
- "活动太火爆"

**处理流程：**
```
检测到限制 → 等待 10 分钟 → 自动重试
           ↓
    再次检测到限制 → 再等待 10 分钟 → 再次重试
           ↓
    达到最大重试次数 → 停止自动化
```

**可配置参数：**
- **等待时间**：默认 600 秒（10 分钟）
- **最大重试次数**：默认 3 次

### 4. 循环轮询设置

**循环次数：**
- 默认 1 次（轮询完所有商品 1 次后停止）
- 设置 0 表示无限循环
- 设置 5 表示轮询 5 轮后停止

**循环间隔：**
- 默认 60 秒
- 每轮循环之间的等待时间

**执行流程示例：**
```
第 1 轮开始
  → 商品 1 → 商品 2 → 商品 3
✓ 第 1 轮完成

⏱️ 等待 60 秒...

第 2 轮开始
  → 商品 1 → 商品 2 → 商品 3
✓ 第 2 轮完成

✅ 完成 2 轮轮询，停止
```

### 5. 悬浮窗快捷控制

**功能：**
- 系统级悬浮窗，显示在所有应用上层
- 一键开启/关闭自动化购买
- 实时显示自动化状态
- 可拖动位置（安卓系统支持）

**界面：**
```
┌─────────────────────────┐
│ 🤖 自动化：关闭         │
│                         │
│ [▶️ 开启自动化]        │
└─────────────────────────┘
```

**开启后：**
```
┌─────────────────────────┐
│ 🤖 自动化：开启         │
│                         │
│ [⏹️ 关闭自动化]        │
└─────────────────────────┘
```

**使用方法：**
1. 在主界面点击"🎛️ 悬浮窗"按钮
2. 悬浮窗显示在屏幕右上角
3. 点击按钮切换自动化状态
4. 状态同步到主应用和 WebView

### 6. 主界面

- **商品列表** - 显示所有已添加的商品 ID
- **添加商品** - 支持单个或批量添加（逗号/换行分隔）
- **查看** - 点击打开商品详情页（WebView 内加载）
- **删除** - 移除商品
- **开始轮询** - 自动循环访问所有商品
- **设置** - 进入设置页面
- **悬浮窗** - 打开/关闭悬浮窗控制

### 7. 设置页面

| 设置项 | 说明 | 默认值 |
|--------|------|--------|
| **账号类型** | 个人号 / 企业号（决定购买流程） | 个人号 |
| **自动化购买** | 开启后点击商品会自动执行购买流程 | 关闭 |
| **限制后等待时间** | 检测到限制后的等待时间（秒） | 600 秒 |
| **最大重试次数** | 限制后最多重试次数 | 3 次 |
| **循环次数** | 轮询循环次数（0=无限） | 1 次 |
| **循环间隔** | 每轮循环之间的间隔（秒） | 60 秒 |
| **轮询间隔** | 每个商品之间的间隔（秒） | 60 秒 |

## APK 打包流程

### 环境要求

- **操作系统**: Linux (推荐 Ubuntu 20.04+) 或 macOS
- **Python**: 3.8+
- **磁盘空间**: 至少 5GB

### 方式 1: GitHub Actions 云打包（推荐）

```bash
cd /home/admin/openclaw/workspace/xianyu-apk

# 关联 GitHub 仓库（替换为你的用户名）
git remote add origin https://github.com/YOUR_USERNAME/xianyu-apk.git

# 推送代码（自动触发打包）
git push -u origin main --tags
```

详见 `GITHUB_ACTIONS_GUIDE.md`

### 方式 2: 本地打包

```bash
# 安装依赖
sudo apt-get install -y python3-pip openjdk-11-jdk autoconf automake build-essential
pip3 install buildozer cython

# 打包
cd xianyu-apk
buildozer -v android debug
```

## ⚠️ 重要警告

### 自动化购买风险

1. **平台风控** - 自动化操作可能触发闲鱼风控系统
2. **账号风险** - 可能导致账号被限制或封禁
3. **财产损失** - 误操作可能导致意外下单

### 使用建议

- ⚠️ **首次使用请先测试** - 用低价商品验证流程
- ⚠️ **开启免密支付需谨慎** - 建议设置支付限额
- ⚠️ **轮询间隔不要太短** - 建议 60 秒以上
- ⚠️ **遵守平台规则** - 请勿用于恶意抢购或黄牛行为

## 注意事项

⚠️ **合规使用**
- 本工具仅用于个人便捷访问
- 自动化购买功能存在风险，请谨慎使用
- 请勿用于批量爬取、恶意抢购等可能违反平台规则的行为

⚠️ **技术限制**
- WebView 仅在安卓环境可用（桌面测试使用模拟模式）
- 悬浮窗需要安卓 `SYSTEM_ALERT_WINDOW` 权限
- 闲鱼可能检测自动化访问，如遇验证码请手动处理
- 部分商品信息需要登录才能查看
- 页面元素选择器可能需要根据闲鱼 UI 更新调整

## 常见问题

**Q: 如何批量添加商品 ID？**
A: 在添加弹窗中，用逗号或换行分隔多个 ID：
```
123456,789012,345678
```
或
```
123456
789012
345678
```

**Q: 悬浮窗怎么用？**
A: 
1. 在主界面点击"🎛️ 悬浮窗"按钮
2. 悬浮窗显示在屏幕右上角
3. 点击按钮切换自动化状态
4. 再次点击按钮关闭悬浮窗

**Q: 悬浮窗权限怎么开启？**
A: 
1. 首次使用会提示需要悬浮窗权限
2. 进入手机设置 → 应用管理 → 闲鱼监控器
3. 找到"悬浮窗"或"显示在其他应用上层"
4. 开启权限

**Q: 自动化购买怎么用？**
A: 
1. 进入设置页面
2. 选择账号类型（个人号/企业号）
3. 开启"自动化购买"开关（或使用悬浮窗）
4. 保存设置
5. 返回主界面点击商品"查看"按钮

**Q: 遇到"无法购买"怎么办？**
A: 应用会自动检测并等待 10 分钟后重试。可在设置中调整等待时间和重试次数。

**Q: 如何重置购买限制状态？**
A: 在主界面点击"重置限制"按钮（如有），或重启应用。

**Q: 轮询间隔设置多少合适？**
A: 建议 60-120 秒，太短可能触发平台风控。

**Q: 商品 ID 从哪里获取？**
A: 从闲鱼分享链接中提取，或从浏览器地址栏获取。

**Q: 数据会丢失吗？**
A: 商品列表自动保存到本地文件，重启应用后自动加载。

**Q: 自动化点击不生效？**
A: 
- 检查闲鱼页面结构是否变化
- 查看控制台日志确认 JS 执行结果
- 可能需要调整按钮选择器

## 技术栈

- **UI 框架**: Kivy
- **WebView**: android.webkit.WebView (通过 pyjnius)
- **URL 检测**: 正则表达式匹配页面类型
- **自动化**: JavaScript 注入 + URL 变化触发
- **限制检测**: JS 文本匹配 + 弹窗检测
- **悬浮窗**: android.view.WindowManager (系统级悬浮窗)
- **打包工具**: Buildozer + python-for-android
