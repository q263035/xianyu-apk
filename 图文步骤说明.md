## 3. 推送代码

### 3.1 解压项目文件

找到收到的 `xianyu-apk-v1.6.0.zip` 文件

```
┌─────────────────────────────────────────────────┐
│  📁 桌面                                        │
│  ├─ 📄 其他文件                                 │
│  ├─ 📦 xianyu-apk-v1.6.0.zip  ← 找到这个       │
│  └─ 📄 其他文件                                 │
└─────────────────────────────────────────────────┘
```

右键 → 解压到当前文件夹

```
┌─────────────────────────────────────────────────┐
│  📁 xianyu-apk/                                 │
│  ├─ 📄 main.py                                  │
│  ├─ 📄 webview.py                               │
│  ├─ 📄 buildozer.spec                           │
│  ├─ 📄 README.md                                │
│  ├─ 📄 图文步骤说明.md                          │
│  └─ 📁 .github/workflows/                       │
│     └─ 📄 build.yml                             │
└─────────────────────────────────────────────────┘
```

### 3.2 打开终端/命令行

**Windows 用户：**
```
方法 1: 
  Win + R → 输入 cmd → 回车

方法 2:
  打开文件夹 → 地址栏输入 cmd → 回车
```

**Mac 用户：**
```
Command + Space → 输入 Terminal → 回车
```

### 3.3 进入项目目录

```bash
# 如果解压在桌面
cd Desktop/xianyu-apk

# 或者
cd 桌面/xianyu-apk
```

**小技巧：** 输入 `cd ` 后，直接把文件夹拖到终端窗口，会自动填入路径。

### 3.4 执行 Git 命令

**复制以下命令，一条一条执行：**

```bash
# 第 1 条：初始化 git
git init
```

看到类似输出表示成功：
```
Initialized empty Git repository in .../.git/
```

```bash
# 第 2 条：添加所有文件
git add .
```

（无输出，正常）

```bash
# 第 3 条：提交
git commit -m "Initial commit: 闲鱼商品监控器 v1.6.0"
```

看到类似输出：
```
[main (root-commit) xxxxxxx] Initial commit
 XX files changed, XXX insertions(+)
```

### 3.5 关联 GitHub 仓库

**重要：替换 YOUR_USERNAME 为你的 GitHub 用户名！**

```bash
# 第 4 条：关联远程仓库
git remote add origin https://github.com/YOUR_USERNAME/xianyu-apk.git
```

**示例：** 如果你的用户名是 `zhangsan`
```bash
git remote add origin https://github.com/zhangsan/xianyu-apk.git
```

### 3.6 获取 GitHub Token

推送时需要 Token（不是登录密码）：

**步骤：**
1. 访问：https://github.com/settings/tokens
2. 点击 **Generate new token (classic)**

```
┌─────────────────────────────────────────────────┐
│  Personal access tokens (classic)               │
│  [ Generate new token (classic) ] ← 点击这个   │
└─────────────────────────────────────────────────┘
```

3. 填写备注：`APK Build`
4. 选择过期时间：`No expiration`
5. 勾选权限：
   - ✅ **repo**（全选）
   - ✅ **workflow**

```
┌─────────────────────────────────────────────────┐
│  New personal access token (classic)            │
│                                                 │
│  Note: APK Build                                │
│  Expiration: No expiration ▼                    │
│                                                 │
│  Select scopes                                   │
│  ☑️ repo  ← 展开全选                            │
│  ☑️ workflow                                     │
│                                                 │
│  [ Generate token ] ← 点击这个                  │
└─────────────────────────────────────────────────┘
```

6. **复制 Token**（一串字符，只显示一次！）

```
┌─────────────────────────────────────────────────┐
│  Your personal access token has been created    │
│                                                 │
│  ┌───────────────────────────────────────────┐ │
│  │ ghp_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx    │ │ ← 复制这个
│  └───────────────────────────────────────────┘ │
│                                                 │
│  ⚠️ 请保存好，关闭后不再显示！                  │
└─────────────────────────────────────────────────┘
```


## 🆘 遇到问题？

### 问题 1：推送失败

```bash
# 检查远程仓库
git remote -v

# 如果错误，重新配置
git remote remove origin
git remote add origin https://github.com/YOUR_USERNAME/xianyu-apk.git
git push -u origin main --tags
```

### 问题 2：Token 无效

重新生成 Token：
1. 访问：https://github.com/settings/tokens
2. 删除旧 Token
3. 生成新 Token（记得勾选 repo 和 workflow）
4. 重新推送

### 问题 3：打包失败

查看 Actions 日志，找到错误原因。常见解决方法：
- 网络问题 → 重试
- 依赖缺失 → 检查 build.yml
