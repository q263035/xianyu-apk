## 📋 设置项说明

| 设置项 | 说明 | 默认值 | 范围 |
|--------|------|--------|------|
| **WebView 状态** | 显示 WebView 是否可用 | 模拟模式 (桌面) / 已启用 (安卓) | - |
| **URL 变化检测** | 自动识别页面类型 | 已启用 | - |
| **购买限制检测** | 检测无法购买提示 | 已启用 | - |
| **账号类型** | 决定购买流程 | 个人号 | 个人号/企业号 |
| **自动化购买** | 开启后自动执行购买 | 关闭 | 开/关 |
| **限制后等待时间** | 检测到限制后的等待时间 | 600 秒 (10 分钟) | ≥60 秒 |
| **最大重试次数** | 限制后最多重试次数 | 3 次 | 1-10 次 |
| **轮询间隔** | 自动轮询的时间间隔 | 60 秒 | ≥10 秒 |

## 🔄 账号类型对比

### 个人号流程
```
📦 商品页 → 点击"立即购买"
         ↓ (延迟 5 秒)
📋 订单页 → 点击"确认购买"
         ↓
💰 付款页 → 点击"免密支付"
```

### 企业号流程
```
📦 商品页 → 点击"立即购买"
         ↓ (延迟 5 秒)
💰 付款页 → 点击"免密支付"
```
（跳过订单确认页）

## ⚠️ 警告信息

设置页面底部显示橙色警告：
```
⚠️ 警告：自动化购买存在风险
   请谨慎使用，建议先测试
```

提醒用户：
1. 自动化操作可能触发平台风控
2. 可能导致账号被限制或封禁
3. 误操作可能导致意外下单
4. 建议先用低价商品测试流程

## 💾 保存验证

点击"保存设置"时会验证：
- 轮询间隔 ≥ 10 秒
- 等待时间 ≥ 60 秒
- 重试次数 1-10 之间

验证失败时显示红色错误提示。

## 📊 数据存储

保存后数据写入 `item_ids.json`：
```json
{
  "item_ids": ["123456", "789012"],
  "interval": 60,
  "account_type": "personal",
  "auto_buy": true,
  "retry_delay": 600,
  "max_retries": 3
}
```
